# 提取字段配置说明 (EXTRACTION_CONFIG.md)

本文件定义了 `homepage-info-extractor` 需执行的标准化提取字段及其对应逻辑。

## 1. 核心提取准则
- **原文提取**：只允许记录官网原文，禁止改写或推测。
- **缺失处理**：找不到原文的字段请**直接留空**。
- **元数据同步**：若“提取文本”留空，则该行对应的“披露页面”和“提取 XPath/定位器”也必须同步留空。只有成功提取到内容的记录才允许标注来源。
- **动态页追溯规则**：若动态加载导致 XPath 无法稳定获取，允许 XPath 留空，但必须填写“披露页面 (URL)”并提供可复核定位器（如 CSS 选择器、文本锚点、区块标题）。

### 2) 一对多字段 (Group 6254 / 6489)
#### 6254 (别名表)
**仅限以下五种分类**，提取时需在“名称类别”列明确标注：
1. **英文全称**
2. **繁体全称**
3. **简体全称**
4. **简称**
5. **曾用名**

#### 6489 (人员表)
全量、穷尽提取官网披露的所有人员（如管理层、董事会、法官、各委员会成员等）。若人员分布在多个子页面（如“常任”与“非常任”），必须逐一遍历，不得遗漏。
