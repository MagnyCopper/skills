# 简历智能筛选 (Resume Screener)

## 概述

本技能根据预定义的岗位招聘要求，对用户提供的简历文件进行**两层筛选**：先做不符合项门控检查（默认通过），再对通过者进行结构化评分和匹配度分析，最终输出 Markdown 格式的筛选报告。

招聘要求预先定义在 `references/job-requirements.md` 中，使用时只需提供简历文件和目标职位名称。评估依赖的院校层级体系定义在 `references/university-tiers.md` 中。

## 适用场景

- **HR 初筛**：从大量简历中快速识别高匹配候选人，缩小面试范围
- **部门经理首轮筛选**：对 HR 推荐的候选人进行技术/专业维度的二次评估
- **批量对比**：同时评估多份简历并给出横向比较和排序建议

## 输入要素

| 要素 | 说明 | 是否必填 |
|------|------|---------|
| 简历文件 | PDF / Word (.docx) / 纯文本 格式的简历，可提供一份或多份 | 必填 |
| 职位名称 | 目标岗位名称，用于匹配 `references/job-requirements.md` 中预定义的要求 | 必填 |
| 额外筛选标准 | 口述的补充筛选条件（如"必须能接受出差"），会叠加到预定义要求之上 | 可选 |

## 工作流

### 第一步：加载基础知识和职位要求

1. 读取用户提供的职位名称
2. 加载 `references/university-tiers.md` 中的院校层级体系（C9/985/211 等）
3. 在 `references/job-requirements.md` 中查找对应职位的预定义要求
4. 如果找到匹配的职位定义，加载该职位的完整要求（岗位职责、加分项、不符合项）
5. 如果未找到匹配的职位定义：
   - 询问用户是否使用通用评估维度（参考 `references/screening-dimensions.md`）
   - 或请用户口述该职位的核心要求，当场构建临时筛选标准
6. 检查用户是否提供了额外的筛选条件，如有则叠加到标准之上

### 第二步：解析简历

对每一份简历文件执行以下分析：

1. **提取核心信息**：
   - 基本信息：姓名、年龄/出生年份、性别、籍贯、联系方式
   - 教育背景：学历、专业、毕业院校、毕业时间、是否全日制
   - 工作经历：公司名称、职位、任职起止时间、工作内容摘要
   - 技能清单：专业技能、工具、语言能力、证书
   - 项目经验：项目名称、角色、技术栈、成果

2. **信息完整性标注**：
   - 标注信息缺失项（如未写明年龄、籍贯、任职时间等）
   - 区分"简历明确写明"和"可推断得出"的信息，这在下一步不符合项检查中至关重要

### 第三步：不符合项检查（默认通过）

这一步是**前置门控**，逻辑为：**默认通过，只有命中不符合项才判定为不通过**。

对照职位定义中的"不符合项"列表，逐项检查：

1. **严格信息来源规则**：
   - 只有简历中**明确写明**的信息才能用于不符合项判定
   - 通过推断得出的信息（如从毕业年份推算年龄）**不能**用于不符合项判定
   - 如果某项不符合项所需信息在简历中未明确体现，该检查项标注为"信息不足，无法判定"，视为**通过**

2. **逐项检查并记录**：
   - 对每一条不符合项，明确记录：检查结果（通过/不通过/信息不足）+ 判定依据（引用简历原文）
   - 任何一条命中"不通过"，该候选人即被标记为"初筛不通过"

3. 常见不符合项类型（具体以各职位定义为准）：
   - 任职稳定性：近 N 年内任职经历数量超限
   - 年龄限制：是否超过年龄上限
   - 地域限制：籍贯是否在限制范围内
   - 学历类型：是否包含专科、自考、非全日制等
   - 院校层级：本科及以上院校是否均低于指定层级（需参照 `references/university-tiers.md`）

### 第四步：逐维度评分（仅对通过初筛的候选人）

对通过不符合项门控的候选人，对照 `references/screening-dimensions.md` 中定义的评估维度，使用 5 分制逐一评分：

1. **硬性条件匹配**（权重 25%）：学历要求、工作年限、必备技能/资格
2. **核心能力评估**（权重 60%）：专业技能匹配度、行业经验、项目经验、职级与职责
3. **潜力与加分项**（权重 15%）：成长轨迹、稳定性、软技能线索
4. **风险标注**：职业空窗期、频繁跳槽、职级异常、描述模糊、信息矛盾

初筛不通过的候选人不进入评分环节，直接在报告中标注不符合原因。

### 第五步：生成筛选报告

使用 `assets/report-template.md` 模板，生成结构化的 Markdown 报告：

1. **报告头部**：筛选概览（职位、简历数量、筛选日期、通过/不通过人数）
2. **推荐排序**：按综合匹配度从高到低排列通过初筛的候选人
3. **逐人详情**：不符合项检查结果 + 维度评分详情
4. **横向对比**：多份简历时提供关键维度的对比表格
5. **最终建议**：明确的推荐/不推荐结论及理由

### 第六步：输出报告

- 在对话中直接展示完整的 Markdown 筛选报告
- 如用户需要，可同时保存为文件（默认路径：当前工作目录下 `resume-screening-report.md`）

## 评分标准

采用 5 分制评分，每个维度独立打分：

| 分数 | 含义 | 说明 |
|------|------|------|
| ⭐ 5 | 完全匹配 | 超出预期，该维度表现突出 |
| ✅ 4 | 良好匹配 | 满足要求且有亮点 |
| 👍 3 | 基本匹配 | 满足最低要求，无明显短板 |
| ⚠️ 2 | 部分匹配 | 有明显不足但尚可考虑 |
| ❌ 1 | 不匹配 | 不满足该维度要求 |

**综合评分计算**：核心能力维度权重 60%，硬性条件 25%，潜力加分项 15%。硬性条件任一项为 ❌(1分) 则综合评定为"不推荐"。

## 输出模板

报告必须遵循 `assets/report-template.md` 中定义的结构，确保每次输出格式一致。模板包含以下章节：

1. 筛选概览
2. 推荐排序（含推荐/待定/不推荐三级分类）
3. 候选人详细评估
4. 横向对比矩阵
5. 筛选建议与下一步行动

## 核心原则

1. **默认通过原则**：候选人默认通过初筛，只有明确命中不符合项才判定不通过
2. **推断信息禁用原则**：推断得出的信息不能用于不符合项的判断。例如：
   - 简历未写年龄 → 不能从毕业年份推断年龄来判定"年龄>30"
   - 简历未写籍贯 → 不能从出生地或手机号推断籍贯
   - 推断信息可以用于加分项评估（不导致淘汰）
3. **客观事实原则**：评估基于简历中的客观事实，避免主观臆断
4. **引用支撑原则**：评分理由必须引用简历中的具体内容，不能空泛评价
5. **缺失即中立原则**：对于简历中未明确说明的信息，标注为"未提及"而非做负面推断
