# 证据链验证方法

> 始终加载。评审任何对象类型都要验证文档里的**事实/数据/外部声明**。
> 核心认识：评审者用**专家方法**（SIFT + Lateral Reading + Reuters 标准），**不要**用教育者方法（CRAAP/PROMPT 表面检查）。

## 1. SIFT Method（核心）

出处：Caulfield, *Web Literacy for Student Fact-Checkers*；Stanford HOIC research。**专业 fact-checker 实际使用的方法**。

| 步骤 | 动作 |
|---|---|
| **S — Stop** | 遇到关键事实/数字/引用时**先停**，不要默认相信。情绪反应（"对！"/"不可能！"）是 warning sign |
| **I — Investigate the source** | 出处是什么？一手/二手/三手？可靠？ |
| **F — Find better coverage** | 其他来源是否一致？是否有更权威来源？主流来源（Reuters/AP/官方统计/学术库）是否覆盖？ |
| **T — Trace claims to origin** | 能追溯原始来源？中间有失真？（学术 → 媒体 → 社交 → 文档）引用的"专家说"真是该专家说的？ |

## 2. Lateral Reading（核心）

出处：Wineburg & McGrew, *Lateral Reading*, [Sage Journals 2019](https://journals.sagepub.com/doi/10.1177/016146811912101102)；[Stanford HICUV](https://purl.stanford.edu/yk133ht8603)。

Stanford HOIC 关键发现：**专业 fact-checker 比历史学家和大学生都快且准**——因为他们**立刻离开原文**，到更宽的网络验证。学生留在原网站读细节（最慢最不准）。

为什么留在原文不准：原文越漂亮越容易相信（确认偏倚）/ 原文"细节"是作者**选择性呈现**的 / 真伪判断需要**外部参照系**。

操作：对关键声明**不要从文档本身判断真假** → **离开文档**用搜索引擎查同一事实的其他来源 → 横向对比多源一致性 → 回到文档标注验证结果。

## 3. CRAAP / PROMPT 表面检查（备选，仅在用户说"快速过一下"时）

出处：[CRAAP Test, CSU Chico](https://library.csuchico.edu/sites/default/files/craap-test.pdf)；[PROMPT, Open University](https://university.open.ac.uk/)。

教育场景常用，**不如 SIFT 严谨**——容易陷入表面检查（"网站看起来正规"）而不做真正网络化验证。

- CRAAP：Currency 时效 / Relevance 相关 / Authority 权威 / Accuracy 准确 / Purpose 目的
- PROMPT：Presentation 呈现 / Relevance 相关 / Objectivity 客观 / Method 方法 / Provenance 出处 / Timeliness 时效

## 4. Reuters / Snopes / PolitiFact 专业标准（参考）

新闻行业标准，比教育场景严格得多。

- **Reuters**：Accuracy first（宁慢勿错）/ Named sources preferred / Cross-checking required（单一来源不够）/ Transparent corrections
- **Snopes**：从 claim 开始 → 联系 knowledgeable parties → 用**非党派原始证据** → 公开所有源 → 接受更正
- **PolitiFact Truth-O-Meter**：用**原始来源**验证（不依赖二次报道）→ 联系领域匹配专家 → 用数据库和已发表研究 → 评级 True → Mostly True → Half True → Mostly False → False → Pants on Fire

## 5. IFCN Code of Principles（自检清单）

出处：[IFCN, Poynter](https://ifcncodeofprinciples.poynter.org/)。

评审者自检：我的评审是否**非党派**？**方法**可追溯（每条发现附原文+方法论）？**局限**公开（自评章节承认盲区）？会**接受更正**（追问机制支持挑战）？

## 证据链验证清单

| # | 检查 | 主用方法 |
|---|---|---|
| 1 | 关键事实/数字是否给了出处？ | SIFT-I |
| 2 | 出处可靠（一手 vs 二手 vs 三手）？ | SIFT-I + Lateral Reading |
| 3 | 多源一致？有更权威来源？ | SIFT-F + Lateral Reading |
| 4 | 能追溯原始来源？有失真？ | SIFT-T |
| 5 | 引用真实存在？（抽查 5-10 条） | Lateral Reading + Reuters |
| 6 | 引用准确转述？未断章取义？ | Lateral Reading |
| 7 | 只引支持性证据？反对观点引了吗？ | Snopes / PolitiFact |
| 8 | 来源有利益冲突？ | Reuters |
| 9 | 来源有更正历史？承认错误？ | IFCN |
| 10 | 评审者自身非党派？方法透明？ | IFCN 自检 |
